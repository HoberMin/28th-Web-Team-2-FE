# 와이어프레임 규격 (초안 · looky.my)

> **목적**: Figma 디자인(`Design` 파일 `296:2109` GUI 작업실)을 읽고 유저 플로우 단위로 정리한 **저충실도 와이어프레임 규격**. wireframe-builder가 이 문서대로 더미 데이터 화면을 만든다.
> **진실 소스 계층**: 정책·엔티티 → `domain.md` · 페이지 스펙 → `product-spec.md` · 결과 알고리즘 → 루트 `기획.md`. 이 문서는 그 위에 **Figma 화면 ↔ 라우트 ↔ 상태** 매핑을 더한다.

## 1. 라우트 맵 — 단일 URL 상태전환 (결정됨)

토큰 URL이 곧 주인공의 공간이고, **시간(24h)·역할·응답 수에 따라 한 URL이 상태 전환**한다 (`domain.md §3` 시간 상태머신). 라우트는 최소로 두고 **뷰는 상태로 분기**.

```
/                          랜딩 / 진입
/onboarding/nickname       닉네임 설정 (주인공 온보딩 1)
/onboarding/survey         자기 설문 8문항 (필수 선행)
   └─ 완료 → [로딩: 링크 발급] → 토큰 생성 → /[token]

/[token]                   단일 URL · 상태머신 (CSR, TanStack Query로 상태 조회)
   분기 키 = (1) localStorage 닉네임 매칭 여부 = 주인공/참여자
            (2) 서버 상태 = 수집중 / 결과 준비 / 미달 / 만료
   ├─ 주인공 · 수집중            → 공유 관리 뷰        (F04)
   ├─ 참여자 · 수집중            → 참여자 진입 → 설문   (F06)
   ├─ 24h 경과 · 응답 3건↑       → [로딩: 결과 생성] → 결과 게이트 → 네컷 (F05)
   ├─ 24h 경과 · 응답 3건 미달    → 재시도/재생성 뷰
   └─ 만료 / 없는 토큰           → 만료·404 안내 뷰
```

- **역할 분기**: 서버 계정 없음 → 주인공은 **기기 localStorage 닉네임**으로만 구분(`domain.md §2`). 매칭 안 되면 참여자.
- **토큰**: 추측 불가능(순번 ID 금지 — `domain.md §3`).
- **메타/OG**: 화면 로직은 클라이언트지만 공유 링크 OG는 `generateMetadata`(서버)로 별도 가능 (`domain.md §3`).
- `/[token]` 내부는 **라우트가 아니라 뷰 컴포넌트**로 분기(ShareView / RespondentSurveyView / ResultView / RetryView / ExpiredView). 설문 진행·결과 스크롤은 각 뷰 내부 상태.

## 3-1. 구현 메모 (초안 1차)

- **디자인 시스템 안에서 구현** — 회색 박스 와이어프레임이 아니라 토큰(`gray/blue/red`)·타이포(`text-head*/body*/caption*`, `font-display1/2`)·`components/ui/button`을 그대로 사용. 이미지·마스코트·GIF·빈 칸 고정이미지 같은 **에셋만 placeholder**(회색 박스).
- **더미 데이터는 최상위 `data/`** (별칭 `@data/*`): `data/questions.ts`(문항 풀), `data/result.ts`(조하리 4칸·결과 더미). 정식 전환 시 이 폴더만 백엔드 응답으로 교체.
- **재사용 컴포넌트**: `src/components/survey/survey-runner.tsx` (본인·참여자 설문 공용), 식별 헬퍼 `src/lib/local-session.ts`.
- **`/[token]` 상태 데모**: 서버 상태가 아직 없으므로 `?view=share|respondent|result|retry|expired` + 화면 상단 **데모 스위처**로 5개 상태 전환. ⚠️ 정식 구현 시 스위처 제거하고 TanStack Query 상태 조회로 대체.
- **병행 세션 소유(미변경)**: 랜딩 `/`(`page.tsx`), 폰트 연결, `components/ui`, 메타데이터. 본 작업은 `/onboarding/survey`·`/[token]`·`/onboarding/nickname`(스텁→기능)만.
